# Twitter OAuth v2 PHP Library Research Notes

## Twitter API v2 OAuth Documentation

### OAuth 2.0 Authentication Methods

1. **Bearer Token (App-Only Authentication)**
   - Authenticates requests on behalf of the developer App
   - Does not involve any users
   - Typically for read-only access to public information
   - Requires passing a Bearer Token with requests
   - Can be generated in the Keys and tokens section of developer Apps
   - Can also be generated by passing API Key and Secret through the POST oauth2/token endpoint
   - Rate limited per endpoint at the App level

2. **OAuth 2.0 Authorization Code Flow with PKCE**
   - Authenticates on behalf of another user
   - Provides more control over application's scopes
   - Improves authorization flows across multiple devices
   - Presents streamlined consent flow for authorizing apps
   - Only displays specific scopes requested from users

### When to Use Bearer Tokens

Bearer Tokens are used for:
- Engagement API
- Account Activity API
- Other APIs that utilize OAuth 2.0 Bearer Token authentication such as v2 and Labs endpoints

### Prerequisites for Bearer Token

- Developer account
- X App
- API keys from the developer portal

### How to Generate a Bearer Token

Using cURL:
```
curl -u "$API_KEY:$API_SECRET_KEY" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

Example with API keys:
```
curl -u "xvz1evFS4wEEPTGEFPHBog:L8qq9PZyRg6ieKGEKhZolGC0vJWLw8iEJ88DRdyOg" \
  --data 'grant_type=client_credentials' \
  'https://api.x.com/oauth2/token'
```

Response format:
```json
{"token_type":"bearer","access_token":"AAAAAAAAAAAAAAAAAAAAML..."}
```

## abraham/twitteroauth Library Structure

The abraham/twitteroauth library is structured as follows:

### Main Files in src/ Directory:
- **TwitterOAuth.php** - Main class for interacting with Twitter API
- **Config.php** - Configuration settings
- **Consumer.php** - OAuth consumer implementation
- **HmacSha1.php** - HMAC-SHA1 signature method
- **Request.php** - HTTP request handling
- **Response.php** - HTTP response handling
- **SignatureMethod.php** - Base signature method interface
- **Token.php** - OAuth token implementation
- **TwitterOAuthException.php** - Custom exception class
- **Util.php** - Utility functions
- **Util/** - Directory with additional utility classes

### Class Hierarchy:
- TwitterOAuth extends Config
- Uses various supporting classes like Consumer, HmacSha1, Response, Token, etc.

### Key Features:
- Namespace: Abraham\TwitterOAuth
- Uses Composer for dependency management
- Implements OAuth 1.0a for Twitter API authentication
- Provides methods for making API requests
- Handles response formatting
- Includes error handling via custom exceptions

### Implementation Notes:
- The library is in maintenance mode with no new feature development planned
- Supports PHP versions listed as "active support" or "security fixes only"
- Uses strict typing (declare(strict_types=1))
- Implements OAuth 1.0a, while our new library will need to implement OAuth 2.0

This research will inform the design of our new Twitter OAuth v2 PHP library, which will need to implement the OAuth 2.0 authentication methods while maintaining a similar structure to the abraham/twitteroauth library.
